Windows Subsystem for Linux (WSL) (Linux 用 Windows サブシステム) は、Windows OS の提供する機能の1つです。この機能により、Windows 上で UNIX 系 OS である Linux を動作させることができます。
Linux はディストリビューションという形態で配布されていて、Ubuntu はその1つです。
この記事は、WSL とその上で動作する Ubuntu のインストールに関する個人的なメモです。


## WSL をインストールすることの利点
ソフトウェア開発のためのツールや、サーバー上で動作するソフトウェアの多くが、Linux 用として提供されています。WSL を使用することにより、これらの開発用ソフトウェアを利用するために別途 Linux 環境を用意することなく、手元の PC で利用することができます。


## インストール環境
この記事では、OS として Windows 11 Home を搭載した PC へのインストールについて述べます。

## インストール
前節に挙げた公式サイトでは、インストール方法として、Power Shell やコマンドプロンプト上でコマンドを実行してインストールする方法が説明されていますが、Windows OS の GUI 操作でインストールする方法もあります。この記事では、GUI 操作による方法について述べます。

また、WSL には WSL 1 と WSL 2 の2つのバージョンが存在しますが、この記事では特に意識しません。

インストールの流れは以下の通りです。

1. WSL の有効化 (Windows の設定変更)
1. ディストリビューション (Ubuntu) のインストール
1. Ubuntu の初回起動

### WSL の有効化 (Windows の設定変更)
まず、Windows OS の設定を変更して、WSL 機能を有効化します。
```
コントロールパネルを起動  
 → プログラムと機能  
 → Windows の機能の有効化または無効化  
 → 「Linux 用 Windows サブシステム」と「仮想マシンプラットフォーム」をチェック  
 → PC を再起動
```
コントロールパネルの起動は、Win キー → 「control」で検索する、などの方法があります。

### ディストリビューション (Ubuntu) のインストール
次に、ディストリビューション (Ubuntu) をインストールします。Ubuntu はストアアプリとしてインストールすることができます。
```
Microsoft Store を起動  
 → 「ubuntu」で検索  
 → 「Ubuntu」、「Ubuntu 24.04.1 LTS」などをインストール
```
Microsoft Store 上で「ubuntu」で検索すると、アプリ名称が「Ubuntu 24.04.1 LTS」のようにバージョン番号の付いているものと、「Ubuntu」のようにバージョン番号の付いていないものがヒットします。バージョン番号なしの「Ubuntu」では、アプリの説明文にインストールされるバージョンが記載されています。


### Ubuntu の初回起動
Windows のスタートメニューに Ubuntu が追加されているので、これを起動します。するとウィンドウが開き、数分待つとユーザー名とパスワードの入力を促すメッセージが表示されるので、これらを入力すれば Ubuntu を使用できる状態になります。

以上で、インストールは完了です。
